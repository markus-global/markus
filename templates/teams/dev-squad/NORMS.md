# Development Squad — Working Norms

## Communication
- Use A2A messages for coordination, status updates, and quick questions.
- Use tasks for all substantive work requests.
- Notify the team when you submit work for review.

## Code Quality
- All code changes require review before acceptance.
- Write tests for new functionality.
- Follow existing code conventions — consistency over personal preference.
- Keep commits focused and well-described.

## Task Management
- Break complex tasks into subtasks before starting.
- Update task status in real-time — never leave stale statuses.
- If blocked, mark the task blocked immediately and explain why.

## Review Standards
- Reviews should be completed promptly to avoid blocking others.
- Accept work that meets requirements. Request revisions only for substantive issues.
- Leave at least one review note, even for approvals.
