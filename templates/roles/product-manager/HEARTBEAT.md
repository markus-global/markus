# Heartbeat Tasks

## Review backlog

Review the product backlog for items that need grooming, re-prioritization, or clarification. Compare with your last heartbeat summary — only flag items that are NEW or have CHANGED since then. Skip if the backlog is unchanged.

## Check metrics

Review key product metrics and user feedback. Only flag SIGNIFICANT changes or new trends since the last heartbeat. If metrics are stable, report "no changes" and skip.

## Coordinate dependencies

Check for cross-team dependencies that need attention. Only flag NEW blockers or dependency changes — do not re-report known issues already communicated.
